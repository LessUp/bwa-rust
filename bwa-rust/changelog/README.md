# Changelog 目录

本目录存放每个版本的**详细变更记录**，与项目根目录的 `CHANGELOG.md` 配合使用。

## 文件命名规范

```
v{major}.{minor}.{patch}.md
```

示例：`v0.1.0.md`、`v0.2.0.md`、`v1.0.0.md`

## 职责分工

| 文件 | 定位 | 内容粒度 |
|------|------|----------|
| `CHANGELOG.md`（项目根） | 汇总入口 | 每版本一段精简摘要 |
| `changelog/vX.Y.Z.md` | 详细记录 | 按阶段/模块组织的完整变更 |

## 文件模板

每个版本文件建议包含以下章节：

```markdown
# Changelog - vX.Y.Z

## 新增功能
- **模块名**：变更说明

## Bug 修复
- **模块名**：修复说明

## 优化
- 优化说明

## 破坏性变更（如有）
- 说明影响范围和迁移方式
```

## 工作流

1. 开发过程中，在对应版本的 `changelog/vX.Y.Z.md` 中追加记录
2. 版本发布时，将摘要同步到 `CHANGELOG.md`
3. `CHANGELOG.md` 的详细记录章节通过链接指向本目录对应文件
