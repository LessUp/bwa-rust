# 索引格式

## 概述

bwa-rust 使用**单文件 FM-index** 格式（`.fm`），将所有索引组件合并到一个可移植文件中。与 BWA 的多文件方式相比，这种设计简化了部署和归档。

## 文件结构

`.fm` 文件使用 [bincode](https://docs.rs/bincode) 序列化，结构如下：

```
┌─────────────────────────────────────────────────────────────┐
│  文件头 (Magic + Version)                                   │
├─────────────────────────────────────────────────────────────┤
│  编码的参考序列文本                                          │
│  (DNA 字母表: {0:$, 1:A, 2:C, 3:G, 4:T, 5:N})              │
├─────────────────────────────────────────────────────────────┤
│  Burrows-Wheeler 变换 (BWT)                                 │
├─────────────────────────────────────────────────────────────┤
│  C 表 (累积字符计数)                                        │
├─────────────────────────────────────────────────────────────┤
│  Occ 采样 (间隔位置的排名统计)                              │
├─────────────────────────────────────────────────────────────┤
│  稀疏后缀数组 (采样 SA 值)                                  │
├─────────────────────────────────────────────────────────────┤
│  Contig 元数据 (名称、长度、偏移量)                         │
└─────────────────────────────────────────────────────────────┘
```

## 关键组件

### Magic Number 和版本号

- **Magic**: `bwa-rust-fm-index`（标识文件类型）
- **Version**: 语义版本，用于兼容性检查

### 编码的参考序列文本

参考序列被连接并使用 3-bit 字母表编码：
- `0`: 哨兵字符 `$`（序列终止符）
- `1-4`: A, C, G, T
- `5`: N（模糊碱基）

### BWT (Burrows-Wheeler 变换)

变换后的文本支持高效的后向搜索。存储：O(n) 字节，n = 参考序列长度。

### C 表

每个字符的累积计数。支持 O(1) 排名查询。

### Occ 采样

以固定间隔存储的排名统计（默认：每 4 个位置）。这种**稀疏采样**相比存储完整排名矩阵减少约 75% 内存。

**权衡**: 查询时间 O(interval) 而非 O(1)，但常量很小，内存节省显著。

### 稀疏后缀数组

只存储每第 k 个 SA 值（默认：k=4）。缺失值使用 BWT 和 Occ 采样按需重建。

## 与 BWA 对比

| 方面 | bwa-rust `.fm` | BWA |
|------|---------------|-----|
| 文件数量 | 1 | 5+ (`.sa`, `.bwt`, `.pac`, `.ann`, `.amb`) |
| 部署 | 单文件复制 | 多文件同步 |
| 归档 | 简单 tar/zip | 必须保留所有文件 |
| 可移植性 | 高 | 中 |
| 索引大小 | 类似 | 类似 |

## 使用方法

### 构建索引

```bash
bwa-rust index reference.fasta -o reference.fm
```

### 使用索引

```bash
bwa-rust align reference.fm reads.fastq -o output.sam
```

::: tip 内存效率

:::
