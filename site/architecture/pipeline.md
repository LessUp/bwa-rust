# 比对流水线

> 从 FASTQ 到 SAM 的完整比对流程。

---

## 详细步骤

### 1. 输入解析

```
FASTQ 文件 → FASTQ Record (name, seq, qual)
                │
                ▼
序列归一化 (ATCGN, 小写转大写)
                │
                ▼
DNA 编码: { A→1, C→2, G→3, T→4, N→5 }
```

### 2. 双向比对

对每个 read 进行两次比对：

- **正向**：原始序列
- **反向互补**：反转 + 碱基配对 (A↔T, C↔G)

### 3. SMEM 种子查找

```
Read:     ACTGACTGACTG
           │      │
           ▼      ▼
         Seed1   Seed2
         (长度10) (长度8)

约束条件:
- min_seed_len: 最小种子长度（默认 19）
- max_occ: 跳过出现次数超过此值的种子（默认 500）
```

### 4. 种子链构建

```
多个种子在参考序列上的位置投影:

参考:  ───────●───────●────●────────
          Seed1   Seed2  Seed3

链评分考虑:
- 种子覆盖范围
- 种子间距离（与 read 长度匹配度）
- 种子密度
```

### 5. Smith-Waterman 扩展

```
种子间隙区域:

参考:  ACGT----ACGT
Read:   CGTACTGAC
       ━━━╋━━━━╋━━━
         Seed  间隙  Seed

带状 SW (带宽 w):
- 仅计算对角线附近 w 范围内的单元格
- 复杂度从 O(n·m) 降到 O(n·w)
```

### 6. 候选管理与输出

```
所有候选比对:
┌─────────┬───────┬─────────┐
│ 得分    │ CIGAR │ 位置    │
├─────────┼───────┼─────────┤
│  58    │ 40M   │ chr1:100│  ← Primary (最佳)
│  45    │ 38M2S │ chr1:200│  ← Secondary
│  38    │ 35M5S │ chr2:500│  ← Secondary
└─────────┴───────┴─────────┘

排序: clip penalty (软剪切惩罚) 升序
输出: max_alignments_per_read (默认 5)
```

## 内存使用模型

| 阶段 | 内存占用 | 说明 |
|------|----------|------|
| 索引加载 | ~5GB | 人类基因组 |
| 每 read | ~10KB | 种子 + 链 + 对齐缓冲区 |
| 并行批次 | ~100MB | 默认 batch_size=1000 |

---

## 性能优化

### 多线程并行

```rust
// 数据并行：每个线程处理一批 reads
reads.par_chunks(batch_size)
    .map(|batch| process_batch(batch, &fm_idx))
    .collect()
```

### 缓冲区复用

```rust
// SW 对齐缓冲区在线程内复用
thread_local! {
    static SW_BUFFER: RefCell<SwBuffer> = ...
}
```

### 稀疏 SA 采样

```
完整 SA: [0, 1, 2, 3, 4, 5, ...]  → 每个位置都存
稀疏 SA: [0, 4, 8, 12, ...]       → 每 4 个位置存一个

查询时通过 LF-mapping 最多跳跃 sa_sample_rate 步
```

## 配置参数

| 参数 | 默认值 | 影响 |
|------|--------|------|
| `threads` | 1 | 并行度 |
| `batch_size` | 1000 | 批处理大小 |
| `band_width` | 16-100 | SW 带宽，影响 indel 敏感度 |
| `min_seed_len` | 19 | 最小种子长度 |
| `max_occ` | 500 | 跳过重复种子，影响速度 |
| `max_chains_per_contig` | 5 | 限制链数量，影响内存 |
| `max_alignments_per_read` | 5 | 输出数量限制 |
